# Att-göra-lista
## Länk till webbplatsen
https://lab2tsjoba1602.netlify.app/

## Hur har jag gått tillväga?
För att skapa denna webbplats började jag till största del med stilningen av statisk data. När jag var nöjd med den gick jag vidare till att skapa ett interface för att-göra-input och började jobba på klassen jag skulle använda.

Klassen hade en tydlig utformningsgrund jag fick arbeta efter. Todos-arrayen är utformad för att lagra alla list-objekt som kommer att skapas och sparas. Det är denna del i klassen som konstant uppdateras och påverkas av de olika metoderna. Konstruktorn initierar denna array genom att ange att den är tom, och loadFromLocalStorage-funktionen fyller arrayen om det finns något att hämta. AddTodo-funktionen validerar input genom att kontrollera att task inte är tomt och att priority har 1,2 eller 3 som inmatat värde. Vid lyckad inmatning skjutsas ny information in i todos-arrayen och saveToLocalStorage-funktionen triggas. MarkTodoCompleted ändras boolean-värdet i ett objekt till true, då standard-värdet anges som false. GetTodos hämtar värdet i todos-arrayen utan att påverka något annat i klassen. SaveToLocalStorage omvandlar objekt till sträng för att lagra i localStorage. LoadFromLocalStorage som aktiveras i konstruktorn hämtar antingen todos från localStorage eller en tom array. Här sker också en omvandling från sträng till objekt, och en uppdatering av todos-arrayen, som senare manipuleras att skrivas ut på webbplatsen.

När klassen var klar kunde den implementeras i koden som styr webbplatsens funktioner. Med hjälp av addTodos regler kunde input valideras, och vid lyckad input sparas innehållet direkt i localStorage då denna funktion automatiskt anropas inom addTodo. En funktion som skapar element och skriver ut list-objekt använder sig av getTodos för att kunna loopa igenom objekten inom todos-arrayen och skriva ut dem om de finns. En annan funktion kunde använda markTodoCompleted för att justera boolean-värdet från false till true. Detta triggades med en eventlyssnare med klick, som skickade med index-värdet för det klickade objektet. Dessa objekt med värdet true tilldelades en annan klass för att avvika utseendemässigt från de uppgifter som inte är slutförda.
